## 1. PURPOSE

The purpose of this policy is to provide guidelines for changes made to YOTTABYTE business processes and information technology, including hardware, software, and data. The intent is to ensure that all changes are controlled, properly prioritized, scheduled, approved, and tested before implementation.

---

## 2. SCOPE

This policy applies to all systems, people, and processes that constitute the YOTTABYTE information systems, including employees and third parties with access.

---

## 3. POLICY STATEMENTS

### 3.1 Documented Operating Procedures
- Procedures must be documented, updated annually, and accessible to authorized personnel.  
- Documentation must be protected from unauthorized access.  
- Stored securely based on classification.  
- System/business owners are responsible.

### 3.2 Capacity Management
- Monitor system resources and forecast future needs.  
- Consider business growth and trends.  
- Pay attention to high-cost/long-lead resources.

### 3.3 Protection from Malware

#### 3.3.1 Controls against Malware
- All systems must be protected from malicious software.

#### 3.3.2 Protection against Malicious Software
- Prohibit unauthorized software  
- Monitor downloads  
- Restrict file usage to business purposes  
- Conduct periodic reviews  
- Update antivirus daily  
- Scan emails and web pages  
- Maintain recovery plans  
- Subscribe to threat alerts  

### 3.4 Threat Intelligence
- Maintain threat intelligence updates and apply based on risk.

### 3.5 Data Loss Prevention
- Install approved antivirus software  
- Implement DLP in O365  

### 3.6 Information Security of Cloud Services
- Maintain SLAs with cloud providers  
- Conduct risk assessments  
- Prevent malware attacks  
- Encrypt data in transit and at rest  

### 3.7 Web Filtering
- Implement web filtering (WAF)  
- Educate employees on threats  

### 3.8 Technical Vulnerability Management
- Perform continuous vulnerability assessments  
- Use approved tools only  

### 3.9 Restrictions on Software Installation
- Only approved software allowed  
- Restrict privileged utilities  
- Maintain vendor-supported versions  

### 3.10 Wireless Access
- Only authorized users allowed  
- Use WPA2 security  
- Co-working provider manages configuration  

### 3.11 Configuration Management
- Maintain baseline configurations  
- Track approvals and changes  
- Harden systems  
- Apply patches regularly  

### 3.12 Security Testing and Audit
- Conduct annual security audits  

### 3.13 Logging and Monitoring

#### 3.13.1 Event Logging
- Log user activities, faults, and security events  
- Monitor admin and user activities  

#### 3.13.2 Protection of Log Information
- Prevent unauthorized log changes  
- Ensure log storage capacity  

#### 3.13.3 Clock Synchronization
- Synchronize systems using NTP  

### 3.14 Network Security
- Protect networks from threats  
- Isolate sensitive systems  
- Control access per policy  
- Restrict personal devices  

#### 3.14.1 Segregation of Networks
- Implement network segregation  
- Maintain network diagrams  

#### 3.14.2 Remote Access (VPN)
- Allow VPN based on role  
- Require approvals  
- Use secure tools  
- Ensure updated antivirus  

### 3.15 Data Backup
- Backup critical data  
- Use Google Drive or encrypted drives  
- Users responsible for backups  

#### 3.15.1 Backup Restoration
- Manager requests restoration  
- IT executes with approval  

#### 3.15.2 Backup Retention
- Store data unless archived by management  

### 3.16 Data Retention

| Data Type | Retention |
|----------|----------|
| Employee Records | 3 years after termination |
| Employee Contracts | Indefinite |
| Client Data | As per contract |
| Vendor Contracts | Indefinite |
| Client Contracts | Indefinite |
| Training Records | 3 years |
| Audit Results | 5 years |
| Visitor Logs | 1 year |
| System Logs | 1 year |
